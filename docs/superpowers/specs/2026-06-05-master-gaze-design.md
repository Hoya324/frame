# 거장의 시선 (Master's Gaze) — Design

**Date:** 2026-06-05
**Status:** Approved design, pending implementation plan

## 1. Purpose

A new editorial section in the FRAME app — a curated, read-and-view collection of
photography's foundational masters and their iconic works. Unlike the rest of the app
(live, crawled, frequently-changing *exhibitions*), this is evergreen curated content:
**master → representative works → per-work commentary**.

For each master the user can:
- See who they are and why they matter (bio, era, nationality).
- View their representative works **with hosted images**.
- Read, for each work, the facts (title, year, medium, holding museum) and an editorial
  note on **why it's a great photograph** and **the context it was made in**.

## 2. Hard constraints

- **Copyright — public-domain / open-access hosting ONLY.** We host images directly only
  when the source museum marks the object CC0 / public domain. No in-copyright works are
  hosted. This naturally scopes the roster to **classic, image-capable masters**
  (≈19th–early-20th century: Eugène Atget, Julia Margaret Cameron, Alfred Stieglitz, the
  US-government FSA photographers Dorothea Lange / Walker Evans, early Japan/Korea
  photography such as Felice Beato · Kusakabe Kimbei, etc.). Modern in-copyright masters
  (森山大道, Cartier-Bresson, 구본창 …) are intentionally **out of scope** for the MVP.
- **Free-tier deployment.** No new paid services. Reuse the existing `GEMINI_API_KEY`
  pipeline. Commentary generation is cached so reruns don't burn quota. Museum APIs used
  must be free and keyless.
- **No risk to the live exhibitions pipeline.** Independent module, independent data file,
  independent build workflow. The existing Sheets/upsert/exhibitions flow is untouched.

## 3. Content generation

- **Facts** (title, year, medium, holding museum, image URL, license/credit) come from the
  museum open-access APIs — auto-filled.
- **Editorial text** (master bio + tagline + per-work "why it's great" commentary) is
  generated by **Gemini as a finished, shippable product** in **ko / en / ja**, with **no
  manual review gate**. (The user explicitly chose "ship the finished version, not a
  draft.") Generated text is cached by master/work id.

## 4. Architecture (chosen: independent pipeline + separate JSON)

```
src/crawler/masters/  →  web/public/data/masters.json  →  Next.js static app (GitHub Pages)
       │
       ├─ curated roster (seed)
       ├─ museum open-access API clients (The Met, AIC, …)
       ├─ public-domain work selection
       └─ Gemini commentary (ko/en/ja, cached)
```

Separate from the exhibitions pipeline because the data shape is fundamentally different
(artist → many works → per-work commentary vs. an exhibition's place/dates/status) and the
content is slow-changing curated material, not a daily crawl.

## 5. Data model — `web/public/data/masters.json`

Follows the existing `tr` convention (locale → field → text). Original text lives in the
flat field + `lang`; the other locales live in `tr`. The web app's existing
`inLocale(original, tr, locale, field)` helper renders it unchanged.

```jsonc
{
  "generated_at": "2026-06-05T00:00:00Z",
  "masters": [
    {
      "id": "eugene-atget",            // stable kebab-case slug
      "name": "Eugène Atget",          // original-language display name
      "lang": "ko",                    // base language of bio/tagline
      "region": "foreign",             // "kr" | "jp" | "foreign"  → list grouping
      "nationality": "FR",             // ISO 3166-1 alpha-2
      "birthYear": 1857,
      "deathYear": 1927,
      "tagline": "파리를 기록한 사진의 선구자",   // one-line hook (base lang)
      "bio": "…왜 이 거장이 중요한지 (Gemini 최종본, base lang)…",
      "portraitUrl": "https://upload.wikimedia.org/…",  // PD portrait, curated in roster (nullable)
      "tr": { "en": { "tagline": "…", "bio": "…" }, "ja": { "tagline": "…", "bio": "…" } },
      "works": [
        {
          "id": "the_met-269725",      // "<source>-<sourceObjectId>"
          "title": "Le Pont Neuf",     // original / museum title (base lang)
          "year": "1900",
          "medium": "Albumen silver print from glass negative",
          "imageUrl": "https://…full…",   // CC0, hosted by the museum
          "thumbUrl": "https://…small…",  // smaller variant for the grid/carousel
          "source": "the_met",            // museum source key
          "sourceUrl": "https://www.metmuseum.org/art/collection/search/269725",
          "credit": "The Metropolitan Museum of Art · CC0",
          "commentary": "…왜 좋은 사진인지 + 맥락 (Gemini 최종본, base lang)…",
          "tr": { "en": { "title": "…", "commentary": "…" }, "ja": { "title": "…", "commentary": "…" } }
        }
      ]
    }
  ]
}
```

Notes:
- `region` drives the 한국 / 일본 / 외국 grouping on the list page.
- `portraitUrl` is a **PD portrait of the photographer**, curated per-master in the roster
  (sourced from Wikimedia Commons), because museum APIs return artworks, not artist
  portraits. Nullable; UI falls back to the master's representative work image.
- Per-work `tr` carries translated `title` + `commentary`. `imageUrl`/`year`/`medium`/
  `credit` are language-neutral.

## 6. Crawler module — `src/crawler/masters/`

- `roster.py` — `ROSTER: list[MasterSeed]`. Each `MasterSeed`: `id`, `name`, `region`,
  `nationality`, `birth_year`, `death_year`, `portrait_url` (curated PD), and a list of
  `sources` describing where to pull works. A source entry is either an **explicit object
  ID list** (hand-picked iconic works — preferred when set) or an **artist query** (museum
  + search term) for auto-pull.
- `museums/base.py` — `MuseumClient` protocol: `fetch_works(query|ids) -> list[RawWork]`,
  where `RawWork` carries title/year/medium/image URLs/source URL/credit and an
  `is_public_domain` flag.
- `museums/the_met.py` — The Met Collection API (free, keyless). `/search?q=&artistOrCulture=true`
  → `/objects/{id}`; keep only `isPublicDomain == true` with a `primaryImage`.
- `museums/aic.py` — Art Institute of Chicago API (free, keyless). Filter
  `is_public_domain == true`; build IIIF image URLs from `image_id`.
- Extensible: LoC (FSA/OWI — US-gov public domain), Smithsonian, Wikimedia can be added
  later behind the same protocol.
- `select.py` — drop non-PD / image-less candidates; if explicit IDs given, use them in
  order; else rank (highlight flag, has-good-image, has-year) and cap at **N ≈ 8–12** per
  master.
- `commentary.py` — Gemini calls for master bio/tagline + per-work commentary, producing
  **ko/en/ja final** text. Reuses the existing Gemini config (`GEMINI_API_KEY`,
  `GEMINI_MODEL`, per-key spacing). Stores results in the cache.
- `cache.py` — JSON cache keyed by master/work id + a content hash of the facts, so
  unchanged entries are never regenerated. `--reset` clears it.
- `build.py` — orchestrate roster → fetch → select → commentary → assemble → write
  `web/public/data/masters.json`.
- **CLI:** `crawler build-masters` (incremental, uses cache) and
  `crawler build-masters --reset` (clear commentary cache + rebuild).

### Roster MVP scale

Target **≈12–20 masters** for the first ship: foreign classic masters as the backbone,
plus a few early Korean/Japanese photography figures where PD images exist. (The earlier
"roster size" question was dismissed; this is the working default and is trivially
adjustable by editing `roster.py`.)

## 7. Web UI

### 7.1 Home carousel (the entry point — `web/src/app/page.tsx`)

The current single featured-exhibition hero (`ExhibitionCardHero`) is replaced by a
**`FeaturedCarousel`** occupying the same hero slot and size.

- **Slides** are a shuffled mix of:
  1. **Featured exhibition** slide(s) — the existing "이달의 전시" hero content
     (`catalog.exhibitions.filter(featured)` / fallback), tap → `/exhibitions/[id]`.
  2. **Master** slide(s) — built from random masters/works in `masters.json`. Each master
     slide is the **same size** as the exhibition hero and shows: the master's **work
     photograph** (background) + the master's **portrait/face** + **name** (+ tagline).
     Tap → `/masters/[id]`.
- **Master slides are randomized** each load so different masters/works keep appearing
  ("거장의 사진은 랜덤하게 계속 바뀌도록").
- **Auto-advance: every 1.4s**, applied to **this carousel only**. Pauses on hover/focus
  and respects `prefers-reduced-motion` (no auto-advance, manual only) for accessibility.
- Swipe mode (the existing `타임/스와이프` toggle) is unchanged; the carousel shows in time
  mode where the hero currently is.
- Component lives at `web/src/components/FeaturedCarousel.tsx` with a `MasterSlide` /
  `ExhibitionSlide` rendering split. Pure, testable slide-list builder
  (`buildCarouselSlides(exhibitions, masters, rng)`) so randomization is unit-testable with
  a seeded RNG.

### 7.2 Masters list — `web/src/app/masters/page.tsx`

Grouped by **한국 / 일본 / 외국** (`region`). Each group is a grid of master cards: portrait
(or representative work) + name + birth–death years + one-line tagline. Tap → detail.

### 7.3 Master detail — `web/src/app/masters/[id]/page.tsx`

- Header: portrait + name + era + nationality + bio.
- Works gallery: each work shows image, title / year / medium, museum credit + source link,
  and the "왜 좋은 사진인지" commentary. Tapping a work opens a larger view with the museum
  original link.
- A "출처 · 라이선스" line per work (credit) keeps attribution explicit.

### 7.4 Catalog loading — `web/src/lib/masters.ts`

Mirror `catalog.ts`: `parseMasters(raw)` + types (`Master`, `MasterWork`, `MastersCatalog`)
+ `loadMasters()` importing `../../public/data/masters.json`. Reuse `inLocale()` /
`localized()` from `catalog.ts` for translations.

### 7.5 i18n

Add `masters.*` keys to all three locale dicts in `web/src/lib/i18n.ts` (section title
"거장의 시선" / "The Master's Gaze" / "巨匠のまなざし", group labels, "why it's great" label,
credit label, etc.).

## 8. Build / deploy wiring

- New GitHub Actions workflow `build-masters.yml`: `workflow_dispatch` + a **monthly cron**
  (content is near-static and regeneration costs Gemini quota). It runs `crawler
  build-masters`, commits the updated `web/public/data/masters.json`, and the existing
  `deploy-pages.yml` ships it with the next deploy.
- Not added to the daily `crawl.yml` (no need; would waste quota).

## 9. Testing

- `tests/fixtures/masters/` — captured Met + AIC API JSON for a couple of masters.
- Unit: museum clients (parse + PD filter), `select` (explicit-ID precedence, ranking,
  cap), `build` (output shape), with Gemini commentary **mocked**.
- Web (vitest): `parseMasters` shape test; `buildCarouselSlides` with seeded RNG
  (mix + randomization); a render test for `FeaturedCarousel` (master slide renders name +
  portrait) and the masters list/detail pages.

## 10. Out of scope (MVP)

- In-copyright / modern masters (text-only or link-only entries).
- A standalone bottom-tab nav entry (entry is the home carousel; a desktop nav link can be
  added later if desired).
- User interactions on masters (scrap/bookmark, search integration).
- Per-work lightbox zoom beyond a simple larger view + source link.
